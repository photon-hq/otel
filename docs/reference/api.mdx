Import everything from the package root:

```ts
import {
  createInstrumentedFetch,
  createLogger,
  getLogLevel,
  instrumentFetch,
  isOtelActive,
  sanitizeEmail,
  sanitizeErrorMessage,
  sanitizePhone,
  sanitizeUrl,
  setLogLevel,
  setupOtel,
  withSpan,
} from "@photon-ai/otel";
```

## `setupOtel(options)`

Boots OpenTelemetry traces and logs.

```ts
function setupOtel(options: SetupOtelOptions): OtelHandle;
```

### `SetupOtelOptions`

```ts
interface SetupOtelOptions {
  serviceName: string;
  serviceVersion?: string;
  endpoint?: string;
  headers?: Record<string, string>;
  resourceAttributes?: Record<string, string | number | boolean>;
  logLevel?: LogLevel;
  instrumentFetch?: boolean | InstrumentFetchOptions;
}
```

### Behavior

- Idempotent within a process.
- Installs async context management.
- Installs W3C trace-context and baggage propagation.
- Configures OTLP/HTTP trace and log exporters when endpoints are available.
- Supports no-endpoint local development.
- Optionally instruments fetch.
- Returns an `OtelHandle`.

## `OtelHandle`

```ts
interface OtelHandle {
  shutdown(): Promise<void>;
}
```

`shutdown()` unpatches fetch instrumentation if setup installed it, shuts down the tracer provider, shuts down the logger provider, and clears the active setup handle.

## `isOtelActive()`

Returns whether `setupOtel()` has an active handle in the current process.

```ts
function isOtelActive(): boolean;
```

This is mainly useful for tests, diagnostics, and cleanup paths.

## `createLogger(module)`

Creates a structured logger.

```ts
function createLogger(module: string): PhotonLogger;
```

### `PhotonLogger`

```ts
interface PhotonLogger {
  debug(message: string, attrs?: LogAttrs, error?: unknown): void;
  info(message: string, attrs?: LogAttrs, error?: unknown): void;
  warn(message: string, attrs?: LogAttrs, error?: unknown): void;
  error(message: string, attrs?: LogAttrs, error?: unknown): void;
}
```

### `LogAttrs`

```ts
type LogAttrs = Record<string, string | number | boolean | undefined>;
```

Undefined values are dropped.

### Behavior

- Emits an OpenTelemetry log record.
- Emits one console line.
- Adds `log.module`.
- Adds user attributes.
- Adds exception attributes when an error is passed.
- Uses active OpenTelemetry context for trace correlation.
- Applies the active log-level gate to both OpenTelemetry and console output.

## `setLogLevel(level)`

Sets the programmatic minimum log level.

```ts
function setLogLevel(level: LogLevel): void;
```

`LOG_LEVEL` still wins when it is set.

## `getLogLevel()`

Returns the current effective log level.

```ts
function getLogLevel(): LogLevel;
```

The value is resolved from `LOG_LEVEL`, then programmatic override, then deployment default.

## `LogLevel`

```ts
type LogLevel = "debug" | "info" | "warn" | "error" | "silent";
```

`silent` suppresses every logger call, including errors.

## `withSpan(name, fn)`

Wraps a sync or async callback in an active span.

```ts
function withSpan<T>(name: string, fn: () => Promise<T> | T): Promise<T>;
```

## `withSpan(name, attrs, fn)`

Wraps a sync or async callback in an active span and attaches attributes.

```ts
function withSpan<T>(
  name: string,
  attrs: LogAttrs,
  fn: () => Promise<T> | T
): Promise<T>;
```

### Behavior

- Starts an active span.
- Sets attributes when provided.
- Marks success as `OK`.
- Records thrown exceptions.
- Sets `error.type` on failures.
- Sanitizes email and phone values in the span status message.
- Ends the span in a `finally` block.
- Rethrows the original error.

If the callback argument is missing, it throws `Error("withSpan: function argument is required")`.

## `instrumentFetch(options)`

Wraps `globalThis.fetch`.

```ts
function instrumentFetch(options?: InstrumentFetchOptions): FetchInstrumentation;
```

### `FetchSpanOptions`

```ts
interface FetchSpanOptions {
  ignore?: (url: string) => boolean;
  attributes?: Attributes;
  redactUrl?: (url: string) => string;
}
```

`ignore` skips spans for matching URLs. `attributes` (the OpenTelemetry `Attributes` type) merges static attributes into every span this instrumentation produces — useful for tagging an SDK's traffic, such as `{ "peer.service": "openai" }`.

`redactUrl` rewrites the URL before it is stored as `url.full`, so you can strip secrets from the query string or path while keeping the span (unlike `ignore`, which drops the span entirely). `server.address` / `server.port` are still derived from the original URL. Pair it with [`sanitizeUrl`](#sanitizeurl-url-options). On Node, setting `redactUrl` forces the `globalThis.fetch` wrap, because the native undici instrumentation has no hook to rewrite `url.full`.

### `InstrumentFetchOptions`

```ts
interface InstrumentFetchOptions extends FetchSpanOptions {
  mode?: "auto" | "global";
}
```

`mode` selects the strategy `setupOtel()` uses: `"auto"` (default) is the native undici instrumentation on Node — when the optional `@opentelemetry/instrumentation-undici` and `@opentelemetry/instrumentation` packages are installed, otherwise the wrap — and the `globalThis.fetch` wrap on Bun; `"global"` forces the wrap on both runtimes. `instrumentFetch()` always performs the wrap regardless of `mode`.

### `FetchInstrumentation`

```ts
interface FetchInstrumentation {
  unpatch(): void;
}
```

### Behavior

- Creates a client span for each non-ignored outbound fetch.
- Names spans by uppercased HTTP method.
- Adds HTTP semantic attributes.
- Injects W3C trace context into outgoing headers.
- Marks `4xx` and `5xx` responses as `ERROR`.
- Records thrown fetch errors.
- Sanitizes thrown error messages before setting span status messages.
- Avoids double wrapping.
- Restores the original fetch through `unpatch()`.

When `setupOtel()` instruments fetch on Node, it uses the native undici instrumentation instead of this wrap — unless `mode: "global"` is set, or the optional undici packages aren't installed, in which case it falls back to this wrap.

## `createInstrumentedFetch(baseFetch, options)`

Returns a new instrumented fetch wrapping `baseFetch` (default `globalThis.fetch`) without mutating the global. Built for SDKs that accept a `fetch` option.

```ts
function createInstrumentedFetch(
  baseFetch?: typeof fetch,
  options?: FetchSpanOptions
): typeof fetch;
```

### Behavior

- Wraps the given fetch so each non-ignored request creates a client span with HTTP attributes and W3C trace-context injection.
- Merges `options.attributes` into every span.
- Applies `options.redactUrl` to the stored `url.full`, if provided.
- Returns the fetch function directly — there is no global lifecycle and no `unpatch()`.
- Is idempotent: passing an already-instrumented fetch returns it unchanged.
- Always uses the wrapper technique, so it works identically on Bun and Node.

On Node, avoid combining this with the global undici instrumentation for the same requests, or they are recorded twice. See [Fetch instrumentation](/guides/fetch-instrumentation).

## `sanitizeEmail(input)`

Masks an email address.

```ts
function sanitizeEmail(input: string): string;
```

Example:

```ts
sanitizeEmail("foo.bar@example.com");
// "fo***@e***.com"
```

## `sanitizePhone(input)`

Masks a phone number.

```ts
function sanitizePhone(input: string): string;
```

Example:

```ts
sanitizePhone("+13315553374");
// "+133xxxx3374"
```

## `sanitizeErrorMessage(input)`

Masks email addresses and phone numbers inside free-form text.

```ts
function sanitizeErrorMessage(input: string): string;
```

Example:

```ts
sanitizeErrorMessage("contact foo.bar@example.com or +13315553374");
// "contact fo***@e***.com or +133xxxx3374"
```

## `sanitizeUrl(url, options)`

Redacts secrets from a URL before it is recorded as a span attribute, following the OpenTelemetry URL semantic conventions: sensitive query-parameter values and `user:pass@` credentials are replaced with the literal `REDACTED`, with the key preserved (`sig=REDACTED`). Non-sensitive parameters and the path are left intact.

```ts
function sanitizeUrl(url: string, options?: SanitizeUrlOptions): string;

interface SanitizeUrlOptions {
  params?: string[];
  redactDefaults?: boolean;
}
```

- `params` — additional query-parameter names to redact, on top of the built-in list. Case-sensitive, matching the semantic conventions.
- `redactDefaults` — redact the built-in semconv list (`X-Amz-Signature`, `X-Amz-Credential`, `X-Amz-Security-Token`, `sig`, `X-Goog-Signature`) and `user:pass@` credentials. Defaults to `true`.

Unparseable input — and input with nothing to redact — is returned unchanged.

It is designed to pair with the `redactUrl` fetch option:

```ts
import { createInstrumentedFetch, sanitizeUrl } from "@photon-ai/otel";

const fetch = createInstrumentedFetch(undefined, {
  redactUrl: (url) => sanitizeUrl(url, { params: ["token", "api_key"] }),
});
```

Example:

```ts
sanitizeUrl("https://api.example.com/v1?token=secret&page=2", {
  params: ["token"],
});
// "https://api.example.com/v1?token=REDACTED&page=2"
```

## `PHOTON_OTEL_VERSION`

Package version constant.

```ts
const PHOTON_OTEL_VERSION: string;
```

It is used as the instrumentation scope version for package-created tracers and loggers.
